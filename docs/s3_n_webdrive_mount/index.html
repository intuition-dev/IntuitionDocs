Setup CDN storage as your HTTP server and mount it

mbake is Cloud v2.0. You do not need to install or maintain any HTTP, DB or any other server.

Steps

  1. Create an account on CDN77 if you don't already have one.

  2. First create a storage for the app, eg: cdn77

  3. Then click CDN link on the top menu.

  4. Scroll to the bottom of page -> click add new SDN storage
  5. To enable a web end point (server), click CDN link on the top menu again.
  6. Click Add new CDN Resource.

  7. On the radio button click storage.

  8. Install on Mac Mountain Duck you may choose a different sftp mount software. There are more than a dozen other choices. Mount replaces FTP.

  9. In Mountain Duck, create a new sftp connection. Fill the fields for server, username with the credentails from your recently created CDN Storage and click by the url field value, the mounted folder will open in Finder.

  10. To verify that the mount is working, you can put an index.html into the mounted folder project root (e.g. W:\wgehner-website) on your file system. You might just copy-paste the index.html you generated here. Then view it in the browser via the Endpoint URL from your recently created CDN Resource.

Summary: With Cloud hosting and mount, you can edit apps from your filesystem and see the edits reflected on the web without extra deployment work.
