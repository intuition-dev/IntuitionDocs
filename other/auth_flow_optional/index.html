Authenthication Flow and Sample App

Basic and Token Auth.

There are several types of auth. Basic is just sending username and password. Token is sending a token, that the server holds instead of username/password. Token can be via query string, header, cookie, etc. Token can comee to any of the distributed servers, so server use a distributed hashmap, such as Redis to store valid tokens. If a Token fails, the user is required to send username/password to the server to get a new token. In general, to be able to do token, you must be comfortable w/ basic, since token auth is just an 'upgrade' and reprsents the already sent username/password.

Example Flow:

  1. User goes to 'secured page''
  2. 'Secured page' reads localy stored cookies
  3. Cookies sent to server/service.
  4. If OK, proceed with 'update', etc.
  5. If not: Go to login page
  6. After login submit, locally store cookie of username/password
  7. Load 'Secured page' / Go to top
